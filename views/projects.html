Tweets by theRealShayon

This was a Data Structures (CSCI 2270) project that a partner, Nick Hanselman and I collaborated to create. the objective was to use any data structure we liked in a creative way. And boy did we get creative. The main program was written in C++ but we used a script written in python to push to Twitter. We used a graph data structure as a Markov Chain to generate sentences based off things our data structures professor said in lecture. First, the program would read a text file in and would store each unique word in the graph. Probabilities of how often each word occurred after it's parent were calculated and used to traverse through the graph using a sudo-random method of finding a path and generating the sentence. It would then output the finished sentence to a file and a script my partner wrote in python using the Twitter API would take the file, read it and Tweet it to the Twitter account we had set up.

The Library Project

This was a project for my Intro to Computing class. THe objective was to create a library system that took in a list of books, authors, and users and put them all into arrays that were all accessible through a menu system made by us. All of the header files and the cpp file were written from scratch. It was written entirely in C++. There isn't a UI for it (yet ;) ) as that was beyond the scope of what we were doing in the class. But at the time, it really challenged my understanding of efficiency, how the code should be structured, and all my fundamentals surrounding arrays, classes, and what data types to use. It's sort of simplistic and yet somewhat complicated considering it was a project for an Intro class.

SmashHub

My team and I made this at the 5th annual HackCU Hackathon. I've been a big fan of Super Smash Bros. for quite some time now. My partner came up with an idea to make some sort of tournament bracket generator. We planned on having different profiles for different people playing and we would store all their stats in JSON files that the application would read on load. We also decided to try something new since it was the Hackathon because that's the time to do it! So we made the application using Electron.js. This posed a few issues as we had never used it before and neither of us had really delved into any sort of JavaScript which would be needed for what we wanted to do. This was solved pretty quick over the course of the event. We hope to implement some sort of database to store all the data at some point as well as clean up the front end, but this was all done in 20 hours!
